# Media Ideation - WEBサイト・メディア企画支援

## Overview

WEBサイトやメディアを作成する前に、目的・ターゲット・コンセプトを対話的に整理し、企画書としてまとめる。

**対象メディア:**
- WEBサイト（コーポレート、LP、ポートフォリオ、ECなど）
- ブログ・オウンドメディア
- SNSアカウント・コンテンツ
- 動画チャンネル・ポッドキャスト
- その他デジタルメディア全般

## The Process

### Phase 1: 目的・ゴール設定

**理解すべきこと:**
- なぜ作るのか？（課題・動機・きっかけ）
- 何を達成したいか？（ビジネス目標・個人目標）
- 成功の定義は？（KPI・指標・期待する成果）
- 制約条件は？（予算・期間・技術スキル・運用体制）

**質問例:**
- 「このサイトを作ろうと思ったきっかけは何ですか？」
- 「半年後、このメディアがどうなっていれば成功ですか？」

### Phase 2: ターゲット分析

**理解すべきこと:**
- 誰に届けたいか？（年齢・職業・興味関心）
- どんな状況で見るか？（デバイス・シーン・時間帯）
- 何を求めているか？（ニーズ・課題・欲求）
- 現状どうしているか？（代替手段・競合利用）

**アウトプット:**
- 簡易ペルソナ（1-2名）
- 利用シーンの具体化

### Phase 3: 競合・参考分析

**実施すること:**
- ユーザーが挙げた参考サイト/メディアの分析
- 競合・類似サービスの特定と比較
- 良い点・改善できる点の整理

**質問例:**
- 「参考にしたいサイトやメディアはありますか？」
- 「これは違うな、と感じるサイトはありますか？」

**ツール活用:**
- WebFetch/WebSearchで参考サイトを調査
- 競合の強み・弱みを整理

### Phase 4: コンセプト設計

**決めること:**
- 独自の価値提案（なぜこのメディアを選ぶのか）
- トーン&マナー（世界観・雰囲気）
- キーメッセージ（一言で伝えるなら）

**提案方法:**
- 2-3のコンセプト案を提示
- 各案のメリット・デメリットを説明
- 推奨案を明示して理由を述べる

### Phase 5: コンテンツ戦略

**設計すること:**
- 必要なページ/コンテンツ一覧
- 情報の優先順位（何をもっとも伝えたいか）
- サイトマップ/コンテンツマップ
- 更新・運用計画（頻度・担当・ネタ出し方法）

**質問例:**
- 「絶対に載せたい情報は何ですか？」
- 「更新はどのくらいの頻度で考えていますか？」

### Phase 6: デザイン方向性

**決めること:**
- ビジュアルの方向性（モダン/クラシック、シンプル/装飾的など）
- カラーイメージ（既存ブランドカラー or 新規）
- 参考デザインの選定
- レスポンシブ要件

**アウトプット:**
- ムードボード的な方向性整理
- 簡易ワイヤーフレーム案（テキストベース）

## Key Principles

- **1メッセージ1質問** - 複数質問で圧倒しない
- **選択肢を提示** - 可能な限り2-4の選択肢を用意
- **具体例で確認** - 抽象的な議論を避け、実例で認識を合わせる
- **段階的に検証** - 各フェーズの終わりに「ここまでで認識合っていますか？」
- **柔軟に戻る** - 後から気づいたことがあれば遠慮なく修正

## Output Format

### 企画書の構成

```markdown
# [プロジェクト名] 企画書

## 1. プロジェクト概要
- 目的・背景
- 成功指標（KPI）
- 制約条件

## 2. ターゲット
- ペルソナ
- 利用シーン
- ニーズ・課題

## 3. コンセプト
- 独自の価値提案
- キーメッセージ
- トーン&マナー

## 4. 競合分析
- 競合・参考サイト
- 差別化ポイント

## 5. コンテンツ設計
- サイトマップ/コンテンツ一覧
- 優先順位
- 運用計画

## 6. デザイン方向性
- ビジュアルイメージ
- カラー・フォント方針
- 参考デザイン

## 7. 次のステップ
- TODO リスト
- スケジュール案
```

### 保存先
- `docs/plans/YYYY-MM-DD-[project-name]-media-plan.md`

## After Planning

**次のアクション提案:**
1. 「この企画書を保存しますか？」
2. 「デザイン制作に進みますか？」→ frontend-design スキル連携
3. 「技術的な実装を検討しますか？」→ brainstorming スキル連携

## Notes

- 技術的な実装詳細は本スキルの範囲外（必要なら brainstorming へ）
- デザイン制作は frontend-design スキルと連携
- ユーザーの言語（日本語/英語）に合わせて対応
